London Cancer Data Observatory

Technical documentation & data source reference
Access the observatory: smileandgiggle.github.io/London-cancer-map

Overview

The London Cancer Data Observatory is an interactive web tool that maps cancer risk indicators, NHS performance and cancer screening uptake across all 33 London boroughs. It supports MP engagement by providing at-a-glance comparisons between boroughs and a one-click briefing document tailored to each constituency's MP.

To use the tool, click any borough on the map to open its data panel on the right. Use the dropdown at the top to recolour the map by a different indicator. To generate a PDF-ready briefing for an MP, click the Briefing button next to their name — this produces a formatted summary including London averages, colour-coded comparisons and CRUK policy asks.

Data Sources

Borough-level indicators — live via API

The following indicators are fetched in real time from the OHID Fingertips API (fingertips.phe.org.uk) each time the tool loads. They will automatically reflect the latest published data without any manual update.

Indicator Source Geography Status
Deprivation (IMD score) OHID Fingertips — indicator 93553 London borough Live API
Smoking prevalence (18+) OHID Fingertips — indicator 92443 London borough Live API
Obesity / overweight prevalence (18+) OHID Fingertips — indicator 93088 London borough Live API

ICB-level indicators — live via API

Cancer outcomes and screening data are not published at borough level by NHS England or OHID. The tool instead uses the nearest available geography: the five NHS Integrated Care Boards (ICBs) covering London. Each borough is assigned to its ICB, and ICB-level data is fetched live from the same Fingertips API.

The five London ICBs are: North Central, North East, North West, South East and South West London.

Indicator Source Geography Status
Cancer incidence rate (per 100,000) OHID Fingertips — indicator 91337 London ICB Live API (ICB)
Bowel cancer screening uptake OHID Fingertips — indicator 92601 London ICB Live API (ICB)
Breast cancer screening uptake OHID Fingertips — indicator 91340 London ICB Live API (ICB)
Cervical cancer screening uptake OHID Fingertips — indicator 93726 London ICB Live API (ICB)
Note on ICB data: Because boroughs share an ICB (e.g. all seven boroughs in North West London display the same ICB figure), these values indicate the broader NHS area's performance rather than the borough's specific outcomes. This is clearly labelled in the tool.

Estimated indicators — not live

Two indicators are not published at borough or ICB level by any national source and are currently shown as estimates based on regional averages. These will not update automatically and should be treated as indicative only.

Indicator Basis Status
Late-stage diagnosis rate Estimate based on national/regional patterns; borough-level data not available via Fingertips or NCRAS public endpoints Estimate
62-day referral-to-treatment standard Estimate; NHS England publishes this at trust level only, not borough or ICB Estimate

MP data — live via Parliament API

MP names, parties and constituencies are fetched from the UK Parliament Members API (members-api.parliament.uk) on load. A static fallback covering all 75 London MPs (as of July 2024) is embedded in the tool in case the API is unavailable. The tool will reflect any by-election results once Parliament's API is updated.

CRUK statistics — manually maintained, no API available

The key facts displayed in the briefing document (“1 in 2 people will be diagnosed with cancer”, “3x more likely to survive if diagnosed at stage 1 vs stage 4”, “360,000 new cases per year”) are drawn from CRUK's published statistics pages. Cancer Research UK does not currently offer a public API for its statistics. These figures are embedded directly in the tool and will need to be manually reviewed and updated if the underlying CRUK data changes.

How often does data refresh?

Live API data (Fingertips and Parliament) is fetched fresh every time someone opens the tool in their browser. The underlying Fingertips datasets are typically updated annually by OHID, so the figures will reflect the most recently published year automatically. There is no scheduled refresh or cache — the tool always requests the latest available data on load.

CRUK statistics and policy asks require a manual edit to the tool's source file if they change. The Influencing Team should review these annually or after any major CRUK statistics publication.
